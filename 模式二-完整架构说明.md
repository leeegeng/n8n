# 模式二：纯 n8n 控制流程 - 完整架构说明

## 一、职责划分

### n8n 负责的功能

| 功能 | 说明 |
|------|------|
| **流程定义** | 在 n8n 中可视化编排完整流程 |
| **条件分支** | 使用 IF/Code 节点实现复杂条件判断 |
| **流转控制** | 决定下一节点、处理驳回逻辑 |
| **任务分配** | 调用本地 API 创建审批任务 |
| **等待审批** | 使用 HTTP Request 节点长轮询等待用户操作 |
| **流程结束** | 调用本地 API 标记流程完成 |

### 本地系统负责的功能

| 功能 | 说明 |
|------|------|
| **数据存储** | 工单、任务、历史记录、通知 |
| **用户认证** | 登录、权限验证 |
| **审批操作** | 用户提交通过/驳回 |
| **通知展示** | 首页显示新消息提醒（铃铛图标） |
| **API 提供** | 供 n8n 调用的回调接口 |

## 二、数据流

```
用户创建工单
    ↓
本地系统: 保存工单数据
    ↓
本地系统: 触发 n8n Webhook (POST /webhook)
    ↓
n8n: 接收数据，开始流程编排
    ↓
n8n: 条件判断（如请假天数>3）
    ↓
n8n: 调用本地API创建审批任务 (POST /create-task)
    ↓
本地系统: 创建任务，给用户发通知（首页铃铛显示红点）
    ↓
n8n: 等待审批 (POST /wait-approval，长轮询)
    ↓
用户: 在本地系统提交审批
    ↓
本地系统: 保存审批结果，唤醒 n8n 等待
    ↓
n8n: 继续执行，判断审批结果
    ↓
n8n: 驳回? → 结束流程 或 回退到指定节点
    ↓
n8n: 通过? → 继续下一节点 或 结束流程
    ↓
n8n: 调用本地API完成流程 (POST /complete-node)
    ↓
本地系统: 更新工单状态，发通知（首页铃铛提示）
```

## 三、n8n 工作流文件

### 1. 请假审批-复杂条件-驳回.json

**功能**：
- 条件判断：病假>3天、事假>1天需要HR审批；>5天需要CEO审批
- 驳回逻辑：经理/CEO驳回直接结束；HR驳回回退到经理

**节点说明**：
```
Webhook → Code(计算条件) → HTTP(创建经理任务) → HTTP(等待审批)
    → IF(是否驳回) → 是 → HTTP(结束驳回)
    → 否 → IF(需要CEO?) → 是 → HTTP(创建CEO任务) → HTTP(等待CEO)
        → IF(CEO驳回?) → 是 → HTTP(结束驳回)
        → 否 → IF(需要HR?) → 是 → HTTP(创建HR任务) → HTTP(等待HR)
            → IF(HR驳回?) → 是 → HTTP(回退经理) / 否 → HTTP(完成)
```

### 2. 会签审批流程.json

**功能**：
- 3人同时审批（总经理、财务经理、技术总监）
- 全部通过才算通过
- 任意1人驳回则整体驳回

## 四、本地系统 API（供 n8n 调用）

### 1. 创建审批任务
```http
POST /api/webhook/n8n/create-task
Headers: X-API-Key: your-api-key

{
  "ticketId": 1,
  "nodeId": "manager_approval",
  "nodeName": "部门经理审批",
  "assigneeType": 2,      // 1-用户, 2-角色, 3-部门
  "assigneeId": 2,        // 角色ID
  "taskType": 1,          // 1-单人, 2-会签
  "assignees": [          // 会签时必填
    {"userId": 1, "userName": "总经理"},
    {"userId": 2, "userName": "财务经理"}
  ],
  "n8nExecutionId": "n8n-exec-123",
  "dueTime": "2024-01-10T18:00:00Z"
}

// 返回
{
  "success": true,
  "targetUsers": [1, 2, 3]  // 收到通知的用户ID列表
}
```

### 2. 等待审批
```http
POST /api/webhook/n8n/wait-approval
Headers: X-API-Key: your-api-key

{
  "ticketId": 1,
  "nodeId": "manager_approval",
  "timeout": 86400  // 秒，默认24小时
}

// 返回: 当用户审批后
{
  "action": "approve",  // 或 "reject"
  "comment": "同意",
  "userId": 3,
  "userName": "张三",
  "timestamp": "2024-01-01T10:00:00Z"
}
```

### 3. 完成节点
```http
POST /api/webhook/n8n/complete-node
Headers: X-API-Key: your-api-key

{
  "ticketId": 1,
  "nodeId": "manager_approval",
  "action": "complete",   // complete, reject
  "comment": "审批完成",
  "nextNode": {           // 驳回到指定节点时必填
    "id": "manager_approval",
    "name": "部门经理审批"
  }
}
```

## 六、配置步骤

### 1. 数据库初始化
```bash
# 执行基础表结构
mysql -u root -p ticket_system < database/schema.sql

# 执行通知表扩展
mysql -u root -p ticket_system < database/notification_schema.sql
```

### 2. 环境变量配置
```env
# .env
DB_HOST=localhost
DB_PORT=3306
DB_NAME=ticket_system
DB_USER=root
DB_PASSWORD=your_password

JWT_SECRET=your_jwt_secret
PORT=3000

# n8n 配置
N8N_WEBHOOK_BASE_URL=http://localhost:5678/webhook
N8N_API_KEY=your_n8n_api_key
APP_URL=http://localhost:3000
```

### 3. 导入 n8n 工作流
1. 在 n8n 中导入 `请假审批-复杂条件-驳回.json`
2. 修改 Webhook 节点的 URL 为实际地址
3. 修改所有 HTTP Request 节点的 `callbackBaseUrl` 为本地系统地址
4. 激活工作流

### 4. 配置本地系统流程
```sql
-- 在 workflow_definitions 表中配置 n8n_config
INSERT INTO workflow_definitions (name, code, status, n8n_config)
VALUES ('请假审批流程', 'leave', 1, '{"webhookUrl": "http://172.16.194.119:5678/webhook/leave-workflow"}');
```

### 5. 启动服务
```bash
# 后端
cd backend
npm install
npm run dev

# 前端
cd frontend
npm install
npm run dev
```

## 七、驳回功能实现

### 直接驳回（结束流程）
```javascript
// n8n 节点配置
{
  "action": "reject",
  "comment": "驳回原因"
  // 不传 nextNode，表示直接结束
}
```

### 驳回到指定节点（重新审批）
```javascript
// n8n 节点配置
{
  "action": "reject",
  "comment": "信息不完整，请补充",
  "nextNode": {
    "id": "manager_approval",
    "name": "部门经理审批"
  }
}
```

本地系统收到后会：
1. 记录驳回历史
2. 更新工单状态为驳回 或 回退到指定节点
3. 创建新的审批任务
4. 发送通知（首页铃铛显示）

## 九、流程示例

### 请假审批流程

**场景**：张三请病假 4 天

1. 张三创建工单 → 本地系统保存 → 触发 n8n
2. n8n 计算：病假 4 天 > 3天 → 需要HR审批
3. n8n 调用 `/create-task` 创建经理审批任务
4. 本地系统通知部门经理（首页铃铛红点）
5. n8n 等待经理审批
6. 经理登录系统 → 看到通知 → 点击通过
7. 本地系统唤醒 n8n，返回审批结果
8. n8n 判断需要CEO审批（>5天？否）→ 需要HR审批（是）
9. n8n 调用 `/create-task` 创建HR审批任务
10. 本地系统通知HR（首页铃铛红点）
11. HR 审批通过
12. n8n 调用 `/complete-node` 结束流程
13. 本地系统通知张三审批完成（首页铃铛提示）

**如果经理驳回**：
- n8n 调用 `/complete-node` 直接结束
- 本地系统通知张三工单被驳回

**如果HR驳回**：
- n8n 调用 `/complete-node` 并指定 `nextNode` 回退到经理
- 本地系统重新创建经理审批任务
- 经理重新审批
