# 模式二架构说明：纯 n8n 控制流程

## 职责划分

### n8n 负责的功能

| 功能 | 说明 |
|------|------|
| **流程定义** | 在 n8n 中可视化编排完整流程 |
| **条件分支** | 使用 IF/Code 节点实现复杂条件判断 |
| **流转控制** | 决定下一节点、处理驳回逻辑 |
| **任务分配** | 调用本地 API 创建审批任务 |
| **等待审批** | 使用 HTTP Request 节点长轮询等待用户操作 |
| **流程结束** | 调用本地 API 标记流程完成 |

### 本地系统负责的功能

| 功能 | 说明 |
|------|------|
| **数据存储** | 工单、任务、历史记录、通知 |
| **用户认证** | 登录、权限验证 |
| **审批操作** | 用户提交通过/驳回 |
| **通知展示** | 首页显示新消息提醒 |
| **API 提供** | 供 n8n 调用的回调接口 |

## 数据流

```
用户创建工单
    ↓
本地系统: 保存工单数据
    ↓
本地系统: 触发 n8n Webhook (POST /webhook)
    ↓
n8n: 接收数据，开始流程编排
    ↓
n8n: 条件判断（如请假天数>3）
    ↓
n8n: 调用本地API创建审批任务 (POST /create-task)
    ↓
本地系统: 创建任务，给用户发通知
    ↓
n8n: 等待审批 (POST /wait-approval，长轮询)
    ↓
用户: 在本地系统提交审批
    ↓
本地系统: 保存审批结果，唤醒 n8n 等待
    ↓
n8n: 继续执行，判断审批结果
    ↓
n8n: 驳回? → 结束流程 或 回退到指定节点
    ↓
n8n: 通过? → 继续下一节点 或 结束流程
    ↓
n8n: 调用本地API完成流程 (POST /complete-node)
    ↓
本地系统: 更新工单状态，发通知
```

## n8n 工作流节点说明

### 1. 请假审批-复杂条件-驳回.json

**流程图**:
```
接收工单 → 计算条件 → 经理审批 → 是否驳回?
                              ↓
                        是 → 结束(驳回)
                        否 → 需要CEO审批?
                                  ↓
                            是 → CEO审批 → CEO驳回?
                                              ↓
                                        是 → 结束(驳回)
                                        否 → 需要HR审批?
                                                  ↓
                                              是 → HR审批 → HR驳回?
                                                              ↓
                                                        是 → 回退经理
                                                        否 → 结束(完成)
                                              否 → 结束(完成)
                            否 → 需要HR审批? → ...
```

**条件逻辑**:
- 病假 > 3天 或 事假 > 1天 → 需要HR审批
- 请假 > 5天 → 需要CEO审批

**驳回逻辑**:
- 经理驳回 → 直接结束
- CEO驳回 → 直接结束
- HR驳回 → 回退到经理重新审批

### 2. 会签审批流程.json

**流程图**:
```
接收工单 → 创建会签任务(3人) → 等待所有人审批 → 统计结果
                                                  ↓
                                        有人驳回? → 是 → 结束(驳回)
                                                    否 → 结束(完成)
```

**会签规则**:
- 3人全部通过才算通过
- 任意1人驳回则整体驳回

## 本地系统 API（供 n8n 调用）

### 1. 创建审批任务
```http
POST /api/webhook/n8n/create-task

{
  "ticketId": 1,
  "nodeId": "manager_approval",
  "nodeName": "部门经理审批",
  "assigneeType": 2,      // 1-用户, 2-角色, 3-部门
  "assigneeId": 2,        // 角色ID
  "taskType": 1,          // 1-单人, 2-会签
  "assignees": [          // 会签时必填
    {"userId": 1, "userName": "总经理"},
    {"userId": 2, "userName": "财务经理"}
  ],
  "n8nExecutionId": "n8n-exec-123",
  "dueTime": "2024-01-10T18:00:00Z"
}
```

### 2. 等待审批
```http
POST /api/webhook/n8n/wait-approval

{
  "ticketId": 1,
  "nodeId": "manager_approval",
  "timeout": 86400  // 秒
}

// 返回: 当用户审批后
{
  "action": "approve",  // 或 "reject"
  "comment": "同意",
  "userId": 3,
  "userName": "张三"
}
```

### 3. 完成节点
```http
POST /api/webhook/n8n/complete-node

{
  "ticketId": 1,
  "nodeId": "manager_approval",
  "action": "complete",   // complete, reject
  "comment": "审批完成",
  "nextNode": {           // 驳回到指定节点时必填
    "id": "manager_approval",
    "name": "部门经理审批"
  }
}
```

## 本地系统页面功能

### 首页通知
- 显示未读通知数量（红点）
- 通知类型：task(审批任务), status(状态变更), error(错误)
- 点击通知跳转到工单详情

### 工单详情页
- 显示工单基本信息
- 显示当前审批节点
- 显示流转历史
- **审批操作区**（如果当前用户有权限）：
  - 通过按钮
  - 驳回按钮
  - 备注输入框

### 我的待办
- 列表显示待审批任务
- 点击进入审批

## 配置步骤

### 1. 导入 n8n 工作流
1. 在 n8n 中导入 `请假审批-复杂条件-驳回.json`
2. 修改 Webhook 节点的 URL 为实际地址
3. 修改 HTTP Request 节点的 `callbackBaseUrl` 为本地系统地址
4. 激活工作流

### 2. 配置本地系统流程
```sql
-- 在 workflow_definitions 表中配置 n8n_config
UPDATE workflow_definitions
SET n8n_config = '{"webhookUrl": "http://localhost:5678/webhook/leave-workflow"}'
WHERE id = 1;
```

### 3. 环境变量
```env
N8N_WEBHOOK_BASE_URL=http://localhost:5678/webhook
APP_URL=http://localhost:3000
```

## 驳回功能实现

### 直接驳回（结束流程）
```javascript
// n8n 节点配置
{
  "action": "reject",
  "comment": "驳回原因"
  // 不传 nextNode，表示直接结束
}
```

### 驳回到指定节点（重新审批）
```javascript
// n8n 节点配置
{
  "action": "reject",
  "comment": "信息不完整，请补充",
  "nextNode": {
    "id": "manager_approval",
    "name": "部门经理审批"
  }
}
```

本地系统收到后会：
1. 记录驳回历史
2. 更新工单状态为驳回 或 回退到指定节点
3. 创建新的审批任务
4. 发送通知
